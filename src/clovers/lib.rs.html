1//! # clovers - ray tracing in rust!
2//!
3//! **Note**: This library is experimental & heavily work in progress. Everything can change at a moment's notice. It is probably not a good idea to use this library for anything other than experimentation for now!
4//!
5//! This project uses GitHub for development and issue tracking. [Link to the repository](https://github.com/Walther/clovers).
6//!
7//! # Guiding thoughts
8//! - Keep it clean: prefer good abstractions, avoid deep integration
9//! - Platform agnostic: hopefully runnable by both CPU and GPU, on desktop and `WebAssembly`, etc
10//! - Prefer correctness: no "cheating" optimizations / approximations
11//! - Look for beautiful light <3
12//!
13//! # How it works
14//!
15//! There are a few core stages of using clovers.
16//!
17//! ## Creating and Loading a Scene
18//!
19//! First, you will need a [Scene](scenes::Scene). You can create a scene manually or utilize [serde](https://docs.serde.rs/) to deserialize from a file. Currently, the example binary uses a JSON format.
20//!
21//! - [Scenes](scenes::Scene) have [Objects](objects::Object)
22//! - [Objects](objects::Object) have a [Material](materials::Material)
23//! - [Materials](materials::Material) usually have a [Texture](textures::Texture)
24//! - Materials and Textures may have unique paramteres to adjust
25//!
26//!
27//! ## Rendering the Scene
28//!
29//! clovers is not opinionated on how you want to render your scene. In a usual scenario, you probably want to have some form of a pixel buffer, with knowledge of the `x` and `y` coordinates of your buffer.
30//!
31//! - Rendering is done by creating [`Ray`](ray::Ray)s and seeing what they hit
32//! - A [`Ray`](ray::Ray) has an origin and a direction
33//! - Every [`Object`](objects::Object) has a `hit()` method that takes a [Ray](ray::Ray) and returns an Option<[`HitRecord`]>
34//! - If you get None, use that information to colorize your pixel with a default color
35//! - If you get Some([`HitRecord`]), use its details to colorize your pixel
36//! - You most likely also want to recurse: depending on the material, maybe `scatter()` and cast a new [`Ray`](ray::Ray)?
37//!
38//! You most likely want to repeat this process multiple times for each of your pixels: generating multiple samples per pixel results in a higher quality image.
39//!
40//!
41//! ## Post processing
42//!
43//! **TODO:** maybe add some post processing utilities?
44//! - denoise support?
45//! - 3D & rendering aware effects?
46//! - etc
47//!
48//! ## Using the result
49//!
50//! At the end, use your pixel buffer - save to an image file, draw a frame in a GUI window, etc.

99/// Internal type alias: this allows the crate to easily switch between float precision without modifying a lot of files.
100pub type Float = f32;
101/// Internal helper: re-exports the pi constant as our internal [Float] type. TODO: selectable at run time instead of build time?
102pub const PI: Float = core::f32::consts::PI;
103/// Internal type alias: a nalgebra [Vector2] which is a vector with two dimensions, containing two of our internal [Float] types
104pub type Vec2 = Vector2<Float>;
105/// Internal type alias: a nalgebra [Vector3] which is a vector with three dimensions, containing three of our internal [Float] types
106pub type Vec3 = Vector3<Float>;
107/// Internal type alias: a nalgebra [Vector4] which is a vector with four dimensions, containing four of our internal [Float] types
108pub type Vec4 = Vector4<Float>;
109/// Internal type alias: a nalgebra [Unit] of a [Vector3]
110pub type Direction = Unit<Vec3>;
111/// Internal type alias: a nalgebra [Vector3]. Intended as a world-space coordinate.
112pub type Position = Vec3;
113/// Internal type alias: a nalgebra [Vector3]. Intended for direction and length between two points, i.e. a direction with a non-unit length
114pub type Displacement = Vec3;
115/// Internal const: epsilon used for avoiding "shadow acne". This is mostly used for the initial minimum distance for ray hits after reflecting or scattering from a surface.
116pub const EPSILON_SHADOW_ACNE: Float = 0.001;
